1
Créer vos Comptes Gratuits
GitHub et Supabase — nécessaires pour mettre le site en ligne
⏱️ 30 min
  • 1
    Créer un compte GitHub (hébergement gratuit)

    Allez sur github.com → Cliquez "Sign up" → Entrez votre email, créez un mot de passe.
    GitHub va héberger votre site gratuitement (comme un serveur gratuit).

  • 2
    Créer un compte Supabase (base de données gratuite)

    Allez sur supabase.com → "Start your project" → Connectez-vous avec votre compte GitHub.
    Supabase va stocker les comptes de vos élèves (emails + mots de passe) de façon sécurisée.

  • 3
    Créer un projet Supabase

    Dans Supabase → "New project" → Choisissez un nom (ex: "espace-cours") → Choisissez une région Europe → Créez un mot de passe fort → Cliquez "Create new project".
    Attendez 1-2 minutes que le projet se crée.

2
Configurer Supabase dans config.js
Connecter votre site à votre base de données
⏱️ 15 min
  • 1
    Récupérer votre clé Supabase

    Dans votre projet Supabase → Cliquez "Settings" (icône engrenage) → "API" → Copiez :
    • Project URL (commence par https://)
    • anon / public key (longue chaîne de lettres)

  • 2
    Ouvrir config.js avec Notepad ou VS Code

    Trouvez le fichier config.js dans votre dossier → Faites clic droit → "Ouvrir avec" → Notepad (ou VS Code).

  • 3
    Remplacez ces 3 lignes dans config.js
    // Avant (à remplacer) :
    const SUPABASE_URL = "https://VOTRE_PROJET.supabase.co";
    const SUPABASE_ANON_KEY = "VOTRE_CLE_ANON_ICI";
    const DEMO_MODE = true;

    // Après (vos vraies valeurs) :
    const SUPABASE_URL = "https://abcdefgh.supabase.co";
    const SUPABASE_ANON_KEY = "eyJhbGc..."; // votre clé longue
    const DEMO_MODE = false; // ← changer true en false
  • 4
    Activer l'authentification dans Supabase

    Dans Supabase → "Authentication" → "Providers" → Assurez-vous que "Email" est activé (toggle vert).

⚠️
Important : Ne partagez jamais votre clé Supabase publiquement. La clé "anon" est faite pour être dans le code client — c'est normal et sécurisé.
3
Mettre le Site en Ligne (GitHub Pages)
Votre site aura une adresse publique gratuite
⏱️ 20 min
  • 1
    Créer un nouveau dépôt GitHub

    Sur github.com → Cliquez le "+" en haut à droite → "New repository".
    Nom : espace-cours → Cochez "Public" → Cliquez "Create repository".

  • 2
    Uploader vos fichiers

    Dans votre dépôt GitHub → Cliquez "uploading an existing file" → Glissez-déposez TOUS les fichiers de votre dossier mon_site_enseignant → Cliquez "Commit changes".

  • 3
    Activer GitHub Pages

    Dans votre dépôt → Onglet "Settings" → Section "Pages" → Source : "Deploy from branch" → Branch : "main" → Cliquez "Save".
    ⏳ Attendez 2-3 minutes → Votre site sera en ligne à :
    https://VOTRE_NOM.github.io/espace-cours/

🎉
Félicitations ! Votre site est maintenant accessible depuis n'importe quel téléphone, tablette ou ordinateur dans le monde !
4
Ajouter vos Élèves
Invitez chaque élève par email — système fermé, sur invitation uniquement
⏱️ 5 min/élève
  • 1
    Aller dans Supabase → Authentication → Users

    Dans votre projet Supabase → Menu gauche : "Authentication" → "Users".

  • 2
    Inviter un élève

    Cliquez "Invite user" → Entrez l'email de l'élève → Cliquez "Send invitation".
    L'élève recevra un email pour créer son mot de passe. Ensuite il pourra se connecter à votre site.

  • 3
    Ajouter le nom dans les métadonnées (optionnel)

    Après avoir invité l'élève → Cliquez sur son email → "Edit user" → Dans "Raw user meta data" ajoutez :
    {"name": "Ahmed Ben Ali", "classe": "6ème A"}

💡
Aucun élève ne peut s'inscrire seul. Seulement les élèves que vous invitez peuvent accéder au site. C'est un système totalement sécurisé.
5
Ajouter vos Vidéos de Cours
Uploadez sur YouTube et ajoutez le lien dans config.js
⏱️ 10 min/vidéo
  • 1
    Enregistrer votre cours

    Utilisez votre téléphone, tablette ou ordinateur pour enregistrer votre cours vidéo.

  • 2
    Uploader sur YouTube comme "Non répertorié"

    Allez sur studio.youtube.com → "Créer" → "Importer une vidéo".
    Dans "Visibilité", choisissez "Non répertorié" (la vidéo ne sera pas trouvable par Google, seulement via votre site).

  • 3
    Copier l'ID YouTube

    Dans l'URL de votre vidéo : https://youtube.com/watch?v=dQw4w9WgXcQ
    Copiez la partie après v= (les lettres en rouge ci-dessus).

  • 4
    Ajouter dans config.js

    Ouvrez config.js → Dans le tableau VIDEOS → Remplacez youtubeId: "" par votre ID :
    Ou ajoutez une nouvelle vidéo en copiant un bloc existant.

    // Exemple d'une vidéo dans config.js :
    {
      id: 10,
      title: "Les Fractions — Exercices",
      subject: "math", // math / arabe / francais / sciences / etc.
      duration: "25 min",
      date: "1 Juin 2026",
      youtubeId:"votre_id_ici", // ← votre ID YouTube
      description:"Description du cours",
      views: 0,
    },
6
Faire un Cours en Direct (Live)
Deux lignes à changer dans config.js avant chaque live
⏱️ 5 min
  • 1
    Démarrer un live YouTube

    Allez sur studio.youtube.com → "Créer" → "Démarrer un direct" → Choisissez "Non répertorié" → Copiez l'ID YouTube du live.

  • 2
    Modifier 3 lignes dans config.js
    const LIVE_NOW = true; // ← changer false en true
    const LIVE_YOUTUBE_ID = "votre_id_live"; // ← ID du live
    const LIVE_SUBJECT = "math"; // ← matière du cours
    const LIVE_TOPIC = "Les Fractions"; // ← thème
  • 3
    Mettre à jour sur GitHub

    Sur github.com → Votre dépôt → Cliquez sur config.js → Icône crayon ✏️ → Modifiez → "Commit changes".
    Le site se met à jour en 2 minutes. Les élèves verront le live automatiquement !

  • 4
    Après le cours, remettre LIVE_NOW à false

    Même procédure → Remettez LIVE_NOW = false dans config.js → Commitez.
